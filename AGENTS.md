# E.O图文

## 项目简介

**E.O图文** 是一个长期维护的个人图文网站，也是 E.O 在互联网上的个人空间。

网站以原创内容为核心，坚持简洁、真实、长期积累的原则，主要展示：

- 摄影作品
- 旅行记录
- 文字作品
- AI工坊创作与实践
- 技术学习过程
- 其他值得长期保存和分享的内容

本项目托管于 GitHub，并通过 GitHub Pages 发布。

---

## 网站目标

本网站不是新闻站，也不是商业网站。

它更像一本不断成长的数字笔记，一座长期建设的个人资料馆。

每一篇文章、每一张图片、每一个工具，都希望能够在多年以后仍然具有阅读和使用价值。

---

## 网站风格

整体设计遵循以下原则：

- 简洁
- 清晰
- 易读
- 内容优先
- 图片优先
- 不追求炫技
- 尽量减少无意义动画
- 优先保证手机和电脑都有良好的浏览体验

配色尽量自然、克制。

字体选择应以阅读舒适为第一目标。

界面默认使用简体中文，同时生成繁体中文版，通过首页设有的开关来切换。

---

## 内容分类

网站内容主要包括以下几个栏目：

### 摄影

摄影作品、器材体验、拍摄心得、图片故事。

### 旅行

旅行见闻、城市记录、酒店、咖啡馆、路线、地图等内容。

### 文字

读书笔记、历史、文学、艺术、人物等主题。

### AI工坊

本人开发的小工具。

包括但不限于：

- 坐标转换工具
- 摄影辅助工具
- 数据处理工具
- AI 小工具

所有工具均尽量采用纯前端实现，方便直接部署到 GitHub Pages。

AI工坊同时记录与 AI 协作完成的软件、网页、图片、研究及实践经验。

重点展示人与 AI 协同工作的真实过程，而不仅是最终成果。

## 写作原则

所有文章应坚持：

- 内容真实
- 尽量准确
- 来源明确（需要时注明）
- 不故意夸张
- 不制造标题党
- 尊重版权
- 尊重事实

对于存在争议的话题，应尽量说明不同观点，而不是只给出单一结论。

---

## 图片原则

图片应优先使用：

- 本人摄影作品
- 已获得授权图片
- 可合法使用素材

图片尽量保留良好的显示质量，同时兼顾网页加载速度。

网页展示图片应经过适当压缩，不直接使用原始照片。

---

## 软件项目原则

所有工具应尽量做到：

- 无需安装
- 浏览器即可运行
- 支持桌面端
- 支持移动端
- 尽量减少第三方依赖
- 保持代码结构清晰

每个工具都应包含：

- 功能说明
- 使用方法
- 更新记录（必要时）

---

## AI 协作原则

AI 是本项目的重要协作者。

AI 的职责包括：

- 协助设计网站
- 协助编写代码
- 协助优化界面
- 协助整理文章
- 协助发现错误
- 协助生成文档
- 协助维护项目结构

但 AI 不应凭空编造事实。

涉及历史、人物、引用、数据等内容时，应尽量进行核实。

## Git 工作流程

本项目会在 Mac Studio 和 MacBook 两台电脑上交替维护。GitHub 上的 `main` 分支是两台电脑之间的唯一共同基准。

每次开始工作时：

1. 先阅读 `README.md` 和 `HANDOFF.md`。
2. 检查当前分支和未提交的修改。
3. 如果工作区干净，先从 GitHub 拉取最新的 `main`，再开始修改。
4. 如果发现未提交修改、分支分叉或拉取冲突，不得直接覆盖、丢弃或强制合并，应先向 Olan 说明情况。

每次完成工作时：

1. 修改项目内容。
2. 本地检查效果。
3. 检查是否影响其他页面。
4. 如果仍有未完成且需要在另一台电脑继续的工作，更新 `HANDOFF.md`；如果已完成，将其恢复为“暂无未完成任务”。
5. 提交 Commit。
6. 推送至 GitHub，确认推送成功后才算完成本次交接。
7. 需要发布时，等待 GitHub Pages 自动发布，再检查线上效果。

Commit Summary 应简洁描述本次修改内容。

不要让两台电脑同时修改同一项内容。离开当前电脑前，应尽量完成提交和推送；换到另一台电脑后，必须先拉取再修改。具体操作见 `docs/MULTI-DEVICE-WORKFLOW.md`。

## 网站长期发展方向

未来网站可以逐步增加：

- 全站搜索
- 标签系统
- 文章分类
- 图片画廊
- 摄影地图
- 软件作品集
- 时间轴
- 多语言支持（视需要）
- RSS
- Sitemap
- SEO 优化

所有新增功能均应保持整体风格统一，不为增加功能而增加复杂度。

---

## 长期理念

E.O图文希望成为一个能够持续维护很多年的个人网站。

它记录的不只是作品，也包括思考、学习和创造的过程。

每一次更新，都应让网站更加完整，而不是更加复杂。
